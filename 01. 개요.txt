가. 정의 및 기능 
	1. 일반적으로 윈도우 프로그램은 MS윈도우 운영체제 기반의 응용프로그램을 말한다.
	2. 윈도우 API는 윈도우 응용 프로그램에서 필요한 기능을 라이브러리 함수 호출 형태로 사용할 수 있게 해준다.
	3. 윈도우 API에서 제공하는 기능은 주로 화면 제어, 그래픽 출력, 통신, 프로세스 관리, 오디오 처리, 보안 등이 있다.
	4. 윈도우 API로 구현한 라이브러리나 헤더 파일 등을 윈도우 SDK(Software Development Kit)라고 한다.

나. 특징
	1. GUI기반 프로그램
		생성한 윈도우에 메뉴, 툴바, 다양한 컨트롤을 표현해 사용자에게 편의성을 제공한다.
	2. 이벤트 기반 프로그램
		사용자가 GUI를 통해 응용 프로그램과 소통할 때 사용자나 또 다른 누군가에 의해 발생하는 사건을 이벤트라고 한다.
		운영체제인 윈도우가 이벤트를 감지하고 메시지를 보내면 윈도우 프로그램은 수신한 메시지에 따라 반응한다.
	3. 멀티스레드 기반 프로그램
		윈도우는 멀티스레드를 이용해 여러가지 작업을 동시에 처리한다.

다. 개발 도구
	1. 많은 IDE가 있지만 내가 사용할 개발도구는 MS에서 만든 Visual Studio이다.
	2. Visual Studio는 많은 버전이 있지만 현재 나온 버전 중 가장 최근 버전인 Visual Studio 2017을 사용할 것이다.

라. 개발 방법(MS Visual Studio 이용)
	1. 프로젝트 생성.
	2. 소스코드 편집 창을 열고 소스 편집.
	3. 컴파일, 디버깅과 실행.

마. 프로젝트 생성법(MS Visual Studio 2017 이용)
	1. 새 프로젝트 생성. (ctrl + shift + n)
	2. Windows 데스크탑 마법사 클릭.
	3. 확인 후 나오는 창에서 '빈 프로젝트' 클릭.
